Creating ePub eBooks from DITA XML

This section explains how to obtain the Dita4Publishers plug-in and create an ePub eBook from content written in DITA XML, with the Open Toolkit.

The prerequisite for this procedure is that you have the Open Toolkit installed and running. No other software is required.

If you have content in DITA XML you can create an ePub eBook just like you would generate a PDF, XHTML or other output. An ePub eBook is a zip file which is renamed with the .epub extension. This file contains XHTML content files and several other files. The whole package must conform with the ePub standard. The plug-in takes care of all of this for you.

  1. Go to Dita4Publishers website where you can find all the links and information you need about the plug-in. The ePub plug-in can be downloaded from the SourceForge project page.
  2. Unzip the downloaded zip file and read the readme file that explains where you should put the plug-in. You should put the contents of the zip in the plug-ins folder of the DITA OT installation.
  3. Run the tests as indicated in the readme file. The code is for processing from the Java command line. If you don't know what this is look it up in the DITA Open Toolkit User Guide. Alternatively, you can use Ant to generate your output just like the other OT test examples. You should now have the test output in the default out folder which is a child of your DITA OT installation. The tests should only take a few seconds to generate. If you want you can make an ePub from some of the other content provided with the OT by pointing the input to a ditamap of choice, for example the Language Reference Guide.
    Note: Generating the Language Reference Guide takes several minutes.
You are now ready to create an ePub eBook from any content you may have in DITA XML, including content you have in foreign languages. Your ePub eBooks can be viewed on an eBook reader device or on your computer with Adobe Digital Editions for example.
