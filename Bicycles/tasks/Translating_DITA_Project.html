Translating a DITA XML Project

Translating a DITA XML project is fairly straightforward if you have a DITA-aware translation memory tool and you keep your files well-organised.

The XML topic files are translated using a DITA-aware translation tool which separates the content and the tags, and validates the translated topic file. The files that need to be translated are:
  • topic files
  • the ditamap
  • header and footer files
Note: Any transforms you created to override the Open Toolkit must not contain any text to be translated.

Translating a topic is fairly straightforward but you must change the xml:lang attribute.

  1. Make a copy of the source language project folder, e.g. the en, and rename it with a language code, e.g. the nl for Dutch. Put this folder at the same level as the source language folder in a parent directory.
  2. Translate the individual topic files and make sure the xml:lang attribute is changed to the target language. This is necessary to ensure the correct localised strings are pulled when the files are processed by the Open Toolkit. If you are using a DITA-aware translation memory tool, you will only see the inline tags that need to be transferred to the target translation.
    If you use a DITA-aware translation memory tool, your target language version will not require any mark-up because you are re-using the mark-up of the original language.
  3. Check that the file is still valid when you have completed the translation.
  4. Close the first file and continue with all the topic files.
  5. When all the topic files are translated, check that the ditamap attribute file has no translatable content, for example in the title attribute.
    Note: It is recommended not to put translatable content in the ditamap.
  6. Translate the header and footer XML files. These files may contain short boilerplate text.
  7. Now copy your build file (an Ant Script for example) and change the input folder and output folder to your new language and build your output again.

You now have your translated content in the same output format or formats that you have your original language. The correct localised strings, links and content are pulled into the output. All the effort you put into creating the transforms for your first language is re-used for all subsequent languages.
